Step 14 — Automate the pipeline

We have now analysed one subject manually. In this step, we create a Bash script that repeats the same workflow automatically for any subject.

1

Why automate the pipeline?

Running FEAT manually is useful when learning the workflow.

But once the analysis works for one subject, we do not want to repeat every step manually for every participant.

Automation allows us to run the same analysis in the same way for each subject.

We first understand the workflow manually, then we automate it.
2

Create the script file

Go to the scripts folder: 

cd ~/fMRI_tutorial_project/scripts

Create a new Bash script: 

vim run_one_subject.sh

Then paste the complete script below into Vim.

3

Copy the automation script

The complete Bash script is provided below.

Copy it, then paste it into the file you created: 

~/fMRI_tutorial_project/scripts/run_one_subject.sh
Open the script file, copy the entire script, then paste it into run_one_subject.sh.

View the complete script →

4

Save and quit Vim

After pasting the script, save and quit Vim: 

Esc
:wq
5

Make the script executable

Run: 

chmod u+x run_one_subject.sh

This allows Ubuntu to execute the file as a program.

6

Run the script for one subject

To prepare the files for subject 051 without launching FEAT: 

./run_one_subject.sh 051

To prepare the files and launch FEAT: 

./run_one_subject.sh 051 run

The word run tells the script to launch FEAT after preparing the files.

Expected result

If everything is correct, FEAT will automatically start and create a new .feat output folder for the selected subject.

7

What does the script do?

Find BOLD Finds the preprocessed MNI BOLD image
Find confounds Finds the fMRIPrep confounds TSV file
Create confounds.txt Extracts selected nuisance regressors
Create EV files Creates Right, Left and Both regressors
Create FSF Adapts the FEAT template for the subject
Run FEAT Launches the first-level analysis
8

Run several subjects

Once the script works for one subject, use a loop to process multiple subjects automatically: 

for sub in 001 003 004 007 008 009 011 012 015 016 017 018 019 021 024 025 030 033 034 037 038 039 046 047 050 051
do
    ./run_one_subject.sh $sub run
done

The same workflow is automatically applied to every subject in the list.

Expected result

FEAT will automatically start for each subject, one after another. Depending on your computer, processing all participants may take a considerable amount of time.

9

Check that the analysis completed successfully

First, go back to the project folder: 

cd ~/fMRI_tutorial_project

Then check that each FEAT output folder was created correctly: 

for sub in 001 003 004 007 008 009 011 012 015 016 017 018 019 021 024 025 030 033 034 037 038 039 046 047 050 051
do
    if [ -d righthanded/bids/derivatives/fsl_output/sub_${sub#0}/sub${sub}_right_left_confounds.feat ]; then
        echo "✅ $sub OK"
    else
        echo "❌ $sub MISSING"
    fi
done

If everything worked correctly, you should obtain one OK line for each subject. 

✅ 001 OK
✅ 003 OK
✅ 004 OK
✅ 007 OK
✅ 008 OK
✅ 009 OK
✅ 011 OK
...
✅ 051 OK
What does this mean?

An OK message means that the FEAT output folder exists for that subject. If you see MISSING, that subject should be checked before continuing.
