Step 10 — Inspect confounds.tsv

Before running FEAT, we inspect the nuisance regressors generated by fMRIPrep.

1

What is the confounds.tsv file?

The confounds.tsv file is generated by fMRIPrep for each functional run.

It contains variables that describe sources of signal variation that are not the main task effect.

These variables are called confounds or nuisance regressors.

2

Why do we use confounds?

fMRI data contain signal changes related to the task, but also signal changes caused by other factors.

For example, head motion, physiological noise and scanner-related fluctuations can affect the BOLD signal.

Adding confounds to the model helps reduce the influence of these unwanted sources of variation.

3

Where is the file?

For subject 051, the confounds file is located in the fMRIPrep derivatives folder.

Path

4

Which confounds are used in this tutorial?

The confounds file contains many possible nuisance regressors.

In this tutorial, we use the same confound strategy for every subject.

Head-motion parameters

These six regressors describe rigid-body translation and rotation of the head.

trans_x trans_y trans_z rot_x rot_y rot_z

Motion derivatives

These regressors represent the temporal change in each motion parameter.

trans_x_derivative1 trans_y_derivative1 trans_z_derivative1 rot_x_derivative1 rot_y_derivative1 rot_z_derivative1

Anatomical CompCor components

Anatomical CompCor components model structured noise estimated from non-grey-matter regions. This tutorial selects the first 13 available components.

a_comp_cor_00 a_comp_cor_01 ... a_comp_cor_12

Motion-outlier regressors

fMRIPrep can create one separate spike regressor for each volume identified as a motion outlier. The number of these columns can differ between subjects.

motion_outlier00 motion_outlier01 ...
Important: framewise_displacement is useful for inspecting head motion, but it is not included as a regressor in the final confounds.txt file used in this tutorial.
