Where does the dataset come from?

The dataset used in this tutorial comes from OpenNeuro, a public platform that hosts openly shared neuroimaging datasets.

The study used here is:

The impact of handedness on the neural correlates during kinesthetic motor imagery: a FMRI study

Its OpenNeuro accession number is:

ds003612

Real OpenNeuro dataset page for ds003612.

2

Why is the dataset version important?

On OpenNeuro, datasets can have several versions.

In this tutorial, the version used is:

1.0.5

Mentioning the version is important for reproducibility, because file content or organisation may change between versions.

How can you download the dataset?

Open the dataset page on OpenNeuro and go to the Download tab.

The simplest option is to use the browser download.

You can also access the dataset directly here: OpenNeuro dataset download page.

OpenNeuro will ask you to choose a local directory where the dataset will be saved.

Once the download starts, leave the page open until it is complete.

4

How should you organise the project locally?

It is important to keep a clean and stable folder structure from the beginning.

In this tutorial, we recommend creating a dedicated project folder and separating the raw BIDS dataset from the preprocessing outputs.

For example: 

fMRI_tutorial_project/
├── bids/
└── derivatives/

The raw BIDS dataset can be stored in bids/.

The outputs generated later by preprocessing tools such as fMRIPrep can be stored in derivatives/.

5

How can you check that the dataset is correctly downloaded?

First, move to the folder where the dataset is stored.

For example:

cd ~/fMRI_tutorial_project/bids

Then use:

ls

You should see the main BIDS files and the subject folders.

You can also use:

find . -name "*events.tsv"

to check that the task event files are present in the dataset. 

lsamson@ubuntu:~$ cd ~/fMRI_tutorial_project/bids

lsamson@ubuntu:~/fMRI_tutorial_project/bids$ ls
