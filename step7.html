Step 7 — Identify the files needed for subject 051

Before building the FEAT model, we need to identify the exact files associated with one subject and one functional run.

Subject 051

Subject ID sub-108TUSW011051
Task execution
Run run-02
events.tsv BOLD confounds.tsv
1

Why do we start with one subject?

Before automating the analysis across multiple participants, it is safer to build and test the workflow on a single subject.

In this tutorial, we begin with subject 051.

This allows us to identify the required files, build the model step by step, inspect the outputs and make sure the analysis is correct before applying it to the rest of the dataset.

2

Which files do we need?

For the first-level analysis, we need several files associated with the same subject and the same run.

  • the events.tsv file from the raw BIDS dataset
  • the preprocessed BOLD image generated by fMRIPrep
  • the confounds.tsv file generated by fMRIPrep

These files describe different parts of the same experiment.

events.tsv task timing
+
BOLD image brain signal
+
confounds.tsv nuisance regressors
→
FEAT model subject-level analysis
3

Where is the events file?

The events file is located in the raw BIDS dataset, inside the functional folder of subject 051.

File

sub-108TUSW011051_ses-1_task-execution_acq-Fs2_run-02_events.tsv

Path

fMRI_tutorial_project/bids/sub-108TUSW011051/ses-1/func/sub-108TUSW011051_ses-1_task-execution_acq-Fs2_run-02_events.tsv

This file contains the timing of the task conditions.

onset when the event starts
duration how long the event lasts
trial_type Right, Left or Both
4

Which preprocessed BOLD file should we use?

fMRIPrep can produce several preprocessed BOLD files for the same run, depending on the output space.

In this tutorial, we use the BOLD image in MNI space.

File

sub-108TUSW011051_ses-1_task-execution_acq-Fs2_run-02_space-MNI152NLin2009cAsym_desc-preproc_bold.nii.gz

Path

fMRI_tutorial_project/bids/derivatives/fmriprep_output/sub-108TUSW011051/ses-1/func/sub-108TUSW011051_ses-1_task-execution_acq-Fs2_run-02_space-MNI152NLin2009cAsym_desc-preproc_bold.nii.gz

This file will be used as the functional input in FEAT.

5

Where is the confounds file?

The confounds file is also generated by fMRIPrep.

It contains nuisance regressors that can be added to the FEAT model to account for signal changes that are not directly related to the task.

File

sub-108TUSW011051_ses-1_task-execution_acq-Fs2_run-02_desc-confounds_timeseries.tsv

Path

fMRI_tutorial_project/bids/derivatives/fmriprep_output/sub-108TUSW011051/ses-1/func/sub-108TUSW011051_ses-1_task-execution_acq-Fs2_run-02_desc-confounds_timeseries.tsv

We will inspect this file in Step 10 before running FEAT.

6

Final check

Before moving on, all files must correspond to the same subject, task and run.

  • sub-108TUSW011051
  • task-execution
  • run-02
These files will be used together to build the subject-level FEAT model.
