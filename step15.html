Step 15 — Group analysis

In this step, we combine the first-level FEAT analyses created in Step 14 to look for activation patterns that are consistent across subjects.

1

What is group analysis?

First-level analysis looks at activation inside one subject.

Group analysis combines the results from all subjects to find effects that are consistent across the group.

This allows us to ask whether a contrast is reliable across participants, instead of looking at only one brain.

First-level analysis = one subject. Group analysis = all subjects together.
2

Which files do we use?

For the group analysis, we do not start again from the original fMRI images.

Instead, we use the .feat folders that were automatically created for each subject by the automation pipeline in Step 14.

Each .feat folder already contains the first-level FEAT results needed for the group analysis.

We use the .feat folders generated by the automation pipeline in Step 14.
3

Open FEAT

Open FEAT from the terminal: 

Feat_gui &

In the FEAT window, change the analysis type to: 

Higher-level analysis
Higher-level analysis is used to combine several first-level FEAT results.
4

Choose the output name

Choose a clear output name for the group analysis. 

~/fMRI_tutorial_project/group_analysis/all_movements

FEAT will automatically create a .gfeat folder after the analysis runs.

Expected output folder:

~/fMRI_tutorial_project/group_analysis/all_movements.gfeat

5

Select the group model

For this tutorial, use a simple group model where each subject contributes equally to the analysis.

In FEAT, set the number of inputs to the number of subject-level .feat folders created by the automation pipeline. 

Number of inputs: 26

Each input corresponds to one participant.

6

Select the FEAT directories

Click Select FEAT directories.

Then select the 26 first-level .feat folders created by the automation pipeline in Step 14.

They are located in: 

~/fMRI_tutorial_project/righthanded/bids/derivatives/fsl_output/

Open each subject folder and select the corresponding .feat directory. Each input must correspond to one subject.

Expected result

Once all the .feat folders have been added, FEAT should display one input directory for each subject.

7

Keep the same contrasts

The group analysis uses the contrasts already defined in the first-level analyses.

In this tutorial, the contrasts correspond to the motor conditions analysed earlier.

Contrast 1 Right hand > Left hand
Contrast 2 Left hand > Right hand
Contrast 3 Right hand > Rest
Contrast 4 Left hand > Rest
Contrast 5 Both hands > Rest
8

Run the group analysis

Ready to run

Once your FEAT window looks like this, click Go.

FEAT will create the group-level analysis folder and compute the results for each contrast.

Expected result

A new .gfeat folder is created inside the group analysis directory.

9

Check that the group analysis finished successfully

When the analysis is finished, verify that all group statistical maps have been created.

Run the following command in the terminal: 

for c in 1 2 3 4 5; do
    if [ -f ~/fMRI_tutorial_project/group_analysis/all_movements.gfeat/cope${c}.feat/stats/zstat1.nii.gz ]; then
        echo "✅ COPE $c completed"
    else
        echo "❌ COPE $c missing"
    fi
done

If the analysis completed correctly, you should obtain: 

✅ COPE 1 completed
✅ COPE 2 completed
✅ COPE 3 completed
✅ COPE 4 completed
✅ COPE 5 completed
Expected result

All five contrasts should be marked as completed. If one contrast is reported as missing, the group analysis did not finish correctly.
