Step 8 — Inspect the events.tsv file

Before creating the EV files, we need to inspect the events table and understand how the task conditions are described.

About this file

File events.tsv
Used for task timing
Important columns onset, duration, trial_type

This file describes when each condition occurred during the run.

1

What is the events.tsv file used for?

The events.tsv file describes the timing of the experimental conditions for one functional run.

In our dataset, it tells us when the subject performed the Right, Left and Both motor conditions during the execution task.

This information is essential because FEAT needs to know when each condition happened in order to model the BOLD response.

events.tsv
→
Right movement Left movement Both hands movement
2

How can we open the file?

1

Find the file

find . -name "*_events.tsv"
2

Move to the directory

cd sub-108TUSW011051/ses-1/func
3

Display its content

cat sub-108TUSW011051_ses-1_task-execution_acq-Fs2_run-02_events.tsv
3

Which columns matter?

For our first-level model, we mainly need three columns.

onset duration trial_type ...
0.000 20.000 Right ...
24.000 20.000 Left ...
48.000 20.000 Both ...
72.000 20.000 Right ...

onset — when the event starts

duration — how long it lasts

trial_type — which condition is performed

4

How should we read one row?

Each row describes one task block.

onset 24.000
duration 20.000
trial_type Left

This means that the Left condition starts 24 seconds after the beginning of the run and lasts for 20 seconds.

5

Why do we need this file?

FEAT does not use the BIDS events.tsv file directly.

We will extract the timing information and create one EV text file for each condition.

events.tsv
→
Right.txt Left.txt Both.txt

In the next step, we will transform the events table into these three EV files so that FEAT can model each condition separately.
