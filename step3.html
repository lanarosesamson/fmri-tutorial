Step 3 — Ubuntu and terminal basics

Before working with the dataset, we need to know how to navigate folders, read file paths and find the files used in the analysis.

If you are already familiar with Linux, you can skip this step. 

lsamson@ubuntu:~$ pwd
/home/lsamson/project_folder

lsamson@ubuntu:~$ ls
bids   scripts   results   README.md

lsamson@ubuntu:~$ mkdir new_folder
lsamson@ubuntu:~$ touch notes.txt
lsamson@ubuntu:~$ vim notes.txt

lsamson@ubuntu:~$ cd bids/

lsamson@ubuntu:~/project_folder/bids$ find . -name "*events.tsv"
./sub-108TUSW011051/ses-1/func/sub-108TUSW011051_ses-1_task-execution_run-02_events.tsv

Why do we need Ubuntu and the terminal?

In this tutorial, most of the analysis is performed in FSL, but the terminal is still essential.

We will use it to move through the project folders, inspect files, create regressors and find the correct BOLD and confounds files.

Being able to read paths and navigate the dataset is therefore a basic but necessary skill for the rest of the pipeline.

▣

What is a path?

What is a path?

A path describes the location of a file or directory in the file system.

When the path points to a file, it includes both the directory and the filename.

For example:

/home/lsamson/project_folder/step3.html

Here, /home/lsamson/project_folder/ is the directory, and step3.html is the filename.

Together, they form the complete file path.

↔

Absolute path vs relative path

Absolute path

An absolute path starts from the root of the file system and gives the complete location of a file or directory.

Example:

/home/lsamson/project_folder/data/events.tsv

Relative path

A relative path starts from the current directory instead of the root of the file system.

The ./ means "start from the current directory".

Example:

./data/events.tsv

Both can point to the same file, but they start from different locations.

▣

Core terminal commands

Command Question Example What it does
pwd How do I know where I am? pwd pwd stands for print working directory. It displays the full path of your current location in the terminal.
ls How do I see what is inside a folder? ls It lists the files and folders contained in the current directory. This is useful to inspect the content of your project folder or to check whether a file is present.
cd How do I move into another folder? cd folder_name cd stands for change directory. For example, if you are in your home folder and want to move into the tutorial folder, you can type: cd project_folder.
mkdir How do I create a new folder? mkdir new_folder mkdir creates a new folder. For example, mkdir results creates a folder called results in your current directory.
touch How do I create a new file? touch notes.txt touch creates a new empty file. For example, touch design.fsf creates a new file called design.fsf.
vim How do I open a file and write inside it? vim notes.txt vim opens a file in the terminal editor. Once the file is open, press i to enter insert mode and start writing inside the file.
:wq How do I save and quit in Vim? :wq In Vim, press Esc first, then type :wq and press Enter. This saves the file and closes Vim.
find How do I find a specific file? find . -name "*events.tsv" This command searches for files or folders inside a given location. The dot . means “search from the current folder”.
cat How do I quickly look at the content of a text file? cat filename For example: cat sub-108TUSW011051_ses-1_task-execution_acq-Fs2_run-02_events.tsv. This prints the content of the file directly in the terminal.
