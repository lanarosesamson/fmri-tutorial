Step 9 — Create the EV files

FEAT needs one explanatory variable file for each condition: Right, Left and Both.

1

What is an EV?

EV stands for Explanatory Variable.

In FEAT, an EV represents one condition of the experiment.

In this analysis, we create one EV for Right, one for Left and one for Both.

2

What format does FEAT need?

FEAT expects a three-column text file.

onset duration amplitude

The amplitude is set to 1 because each event is modelled with the same weight.

3

Create the Right EV

This command keeps only the rows where the condition is Right. 

awk '{gsub(/\r/,""); if ($3=="Right") print $1 "\t" $2 "\t1"}' events.tsv > right.txt

The output file is:

right.txt

4

Create the Left EV

This command keeps only the rows where the condition is Left. 

awk '{gsub(/\r/,""); if ($3=="Left") print $1 "\t" $2 "\t1"}' events.tsv > left.txt

The output file is:

left.txt

5

Create the Both EV

This command keeps only the rows where the condition is Both. 

awk '{gsub(/\r/,""); if ($3=="Both") print $1 "\t" $2 "\t1"}' events.tsv > both.txt

The output file is:

both.txt

6

Check the files

Use cat to check that each EV file contains three columns. 

cat right.txt
cat left.txt
cat both.txt

Each line should contain an onset, a duration and the value 1.

1 8 9 10 16
