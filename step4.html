Step 4 — Understand BIDS

Before locating the files we need, we must understand how the dataset is organised.

1

What is BIDS?

BIDS stands for Brain Imaging Data Structure.

It is a standard way of organising neuroimaging datasets.

Instead of storing files in arbitrary folders with arbitrary names, BIDS defines a predictable structure for subjects, sessions, modalities and metadata.

This makes datasets easier to read, easier to share and easier to analyse with tools such as fMRIPrep or FSL.

Key idea

With BIDS, we can predict where files are stored and what information their filenames contain.

2

Why is BIDS useful?

In practice, BIDS helps us know where to look for the files we need.

If a dataset follows the BIDS standard, we can usually predict:

  • where the anatomical images are stored
  • where the functional runs are stored
  • where the events files are stored
  • how subjects and sessions are named

This is especially useful when working on a multi-subject dataset, because the same structure is repeated across participants.

3

How is a BIDS dataset organised?

A BIDS dataset is usually organised by subject, and sometimes by session.

A typical structure looks like this: 

dataset/
└── sub-051/
    └── ses-1/
        ├── anat/
        └── func/

In our dataset, the subject folders follow the same logic, but with longer identifiers such as:

sub-108TUSW011051

Inside a subject folder, the func/ directory contains the functional runs and their associated task files.

4

How do BIDS filenames work?

BIDS filenames are built from several pieces of information separated by underscores.

For example:

sub-108TUSW011051_ses-1_task-execution_acq-Fs2_run-02_events.tsv

This filename tells us:

  • sub-108TUSW011051 → the subject identifier
  • ses-1 → the session
  • task-execution → the task performed during the run
  • acq-Fs2 → acquisition label
  • run-02 → run number 2
  • events.tsv → the tab-separated file containing the task events
5

Which BIDS files matter for this tutorial?

During this tutorial, we will learn how to locate the files needed for an fMRI analysis.

Depending on the analysis step, we will need to find one file at a time:

  • the anatomical T1 image file
  • the functional BOLD image file
  • the corresponding events.tsv file

To find any of these files, we always follow the same three steps:

  • identify the correct subject folder
  • identify the correct functional run
  • locate the file associated with that run
